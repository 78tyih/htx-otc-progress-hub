## 1. 页面结构

整页为**单栏纵向流式布局**，自上而下 6 个结构带，无任何侧边栏：

| 顺序 | 结构带 | 内容 |
|---|---|---|
| ① | 顶部导航栏（通栏深色条） | 左侧：汉堡菜单按钮 + 大标题 `Overview` + 副标题 `24 open jobs · 4 crews in the field today`；右侧：`⌘K Command` 按钮、`Search jobs, crews.` 搜索框、`YB` 黄色用户按钮、黄色 `New job` 主按钮 |
| ② | KPI 卡片区 | 4 张等宽卡片横排：Open jobs 24 / Crew hours this week 1,842 / Material spend $48.2k / Avg. margin 18.6%，每张含绿色或红色趋势文字（`+3 vs last week`、`+12% vs last week`、`−4% vs last week`、`+1.2pt vs last week`） |
| ③ | 主图表区（整宽面板） | 大号黄色堆叠柱状图：标题 `1,842 hrs` + 副标题 `Crew hours this week`，右上角绿色 `↑ 12%` 徽章；X 轴周一至周日；图例 Billable（黄）/ Overtime（近黑） |
| ④ | 次图表区（整宽面板） | 面积趋势图：标题 `18.6%` + 副标题 `Average job margin`，右上角绿色 `↑ 1.2pt` 徽章；X 轴 W1–W6，18% 处有一条浅色虚线目标线 |
| ⑤ | 三联面板区 | 三等分横排：Job status mix 环形图（中心 42% In progress）/ Material spend by category 横向条形图（带红色 `↓ 4%` 徽章）/ Crew utilization 进度条组（92 / 88 / 76 / 64%） |
| ⑥ | 底部表格区（整宽面板） | `Active jobs` 表格：标题 + 右侧 `All statuses` 筛选框与 `Export` 按钮；5 行数据，状态徽章 In progress 黄 / Review 橙 / Complete 绿 / Blocked 红；页脚有悬浮 `Application ⌄` 胶囊与右下角圆形 `i` 按钮 |

**要点**：先总（KPI）→ 再分（主图表）→ 再拆维度（三联面板）→ 最后落到明细（表格）。每一"带"之间以统一的大间距分隔，结构完全靠间距 + 面板底色区分，没有多余装饰线。

## 2. 信息层级

三级信息层级非常清晰：

1. **一级（结论层）**：KPI 大数字、图表区大标题数字（`1,842 hrs`、`18.6%`、`$48.2k`、`42%`）——白色粗体超大字号，一屏之内第一眼必达。
2. **二级（趋势层）**：绿色/红色趋势文字与右上角趋势徽章（`↑ 12%`、`↑ 1.2pt`、`↓ 4%`）——告诉读者"变好还是变坏"，用色彩直接编码方向，不需要读数字就能感知。
3. **三级（明细层）**：轴标签、图例、表格行、副标题——全部压成 muted 灰色，退居背景。

层级控制手段：**字号差（超大数字 vs 小号灰字）+ 色彩差（白/黄/绿/红 vs 灰）+ 位置（徽章固定右上、明细固定底部）**。每个面板都遵循同一模板：大数字结论在左上、趋势徽章在右上、图表主体居中、图例/轴信息沉底。

## 3. 色彩系统

典型深色运营指挥台配色，**低饱和底色 + 一个高亮主色 + 语义色点缀**：

| 角色 | 截图中的表现 | 目估色值 |
|---|---|---|
| 页面底色 | 近黑 | ≈ `#111111` |
| 面板/导航条底色 | 比页面略亮的深灰 | ≈ `#1a1a1a`–`#1f1f1f` |
| 输入框/深色条形图填充 | 更深的近黑（面板内的"凹下去"层） | ≈ `#141414` |
| 主文本 | 纯白 | `#ffffff` |
| 次要文本（副标题、轴标签、图例、表头） | 中灰 | ≈ `#9b9b9b` |
| **主强调色（亮黄）** | 柱状图主填充、面积图描线、进度条填充、`YB` / `New job` 按钮、`In progress` 徽章、环形图 In progress 段 | ≈ `#ffe000` |
| 成功/正向（绿） | 趋势文字、`↑ 12%` / `↑ 1.2pt` 徽章、`Complete` 徽章、环形图 Complete 段 | ≈ `#10b981` |
| 风险/阻塞（红/品红） | `−4%` 趋势文字、`↓ 4%` 徽章、`Blocked` 徽章、环形图 Blocked 段 | ≈ `#e11d48`–`#ff4d6d` |
| 警示/评审（橙） | `Review` 徽章、环形图 In review 段 | ≈ `#f97316` |
| 网格线/分隔线 | 深灰虚线、表格行细分隔线 | ≈ `#333333` |

**关键手法**：
- 黄色是全页唯一高饱和强调色，承担"主数据"和"主行动"双重身份，视觉焦点被严格控制在一个颜色上。
- 绿/红/橙只做语义信号（趋势方向、状态分类），用量极小，绝不铺大面积。
- 深色界面里反向使用"近黑填充"（Material spend 横向条形图用近黑条 + 灰标签），形成面板内的第二层深浅对比。

## 4. 字体层级

全页单一无衬线字体（Inter 风格），靠字重 + 字号 + 颜色分层：

| 层级 | 用途 | 表现 |
|---|---|---|
| H0 超大数字 | KPI 数值、图表大标题（`24`、`1,842`、`$48.2k`、`18.6%`、`42%`） | 白色、Bold/ExtraBold，约 48–64px（相对 1750 宽画布） |
| H1 页面标题 | `Overview` | 白色 Bold，明显大于面板标题 |
| H2 面板/区块标题 | `Active jobs`、`Job status mix`、`Crew utilization` | 白色 Medium/Bold，中等字号 |
| 副标题/说明 | `24 open jobs · 4 crews in the field today`、`Crew hours this week` | muted 灰，常规字重 |
| 表格列头 | `ID / JOB / FOREMAN / STATUS / MARGIN` | muted 灰、小号、大写字母、加宽字距 |
| 表格内容 | 工单单号/项目名（大写灰）、人名/数字（白） | 同行内用灰白对比区分主次 |
| 趋势文字 | `+3 vs last week` 等 | 绿/红，小号，常规字重 |
| 徽章文字 | `↑ 12%`、`In progress` 等 | 徽章底色上的高对比文字（黄徽章用黑字，绿/红/橙徽章用白字） |

**要点**：数字永远比文字大至少两个级别；说明性文字一律灰色降噪；表格内通过"灰大写（编号/项目名） vs 白（人名/数值）"做单行内的二次分层。

## 5. 卡片样式

- **KPI 卡片**：深灰面板底（≈`#1f1f1f`），无描边或极弱描边，圆角很小（≈4–8px），内边距充裕。内部固定三段式：顶部灰色小标签 → 中部白色超大数字 → 底部绿/红趋势文字。4 张等宽，间距均分。
- **图表面板**：与 KPI 卡同族，但为整宽或三等分宽；标题区（大数字 + 副标题）固定在面板左上，徽章固定右上。
- **统一特征**：卡片之间靠**等宽间距**分隔而非边框；所有卡片同底色、同圆角、同内边距逻辑，仅宽度不同 —— 一套面板组件的三种宽度变体（1/4、1/3、整宽）。

## 6. 数据可视化方式

| 图表 | 做法 | 设计细节 |
|---|---|---|
| 堆叠柱状图（主图） | 亮黄（Billable）+ 近黑（Overtime）纵向堆叠，周一至周日 7 根 | Y 轴 0/150/300/450/500 灰字 + 深灰虚线网格；柱体无描边、小圆角；底部居中图例（色块 + 灰字） |
| 面积趋势图 | 黄色描线 + 黄色低透明渐变填充，W1–W6 六个数据点带圆点 | 18% 处一条**浅色虚线目标线**横贯全图 —— 目标参照线是"进度看板"的关键元素；Y 轴 14%–22% |
| 环形图（Donut） | 中心超大 `42%` + 下方灰色 `In progress`；右侧图例四行：黄 In progress 42% / 橙 In review 18% / 绿 Complete 31% / 红 Blocked 9% | 粗环段、段间有间隙；只高亮最大占比项于中心 |
| 横向条形图 | Lumber / Steel / Fixtures / Fasteners / Other 五类，**近黑条 + 面板底对比**，X 轴 `$0k–$16k` | 深色界面里的"负形"条形：条比面板更暗而非更亮 |
| 进度条组 | Crew 1–4，黄色填充条 + 深灰轨道，右侧白色粗体百分比（92/88/76/64%） | 标签在左上、数值在右上、条在下，四行等距重复 |
| 趋势徽章 | 图表右上角固定位置：绿底白字 `↑ 12%`、`↑ 1.2pt`，红底白字 `↓ 4%` | 方向箭头 + 数值，看一眼即知涨跌 |

**通用手法**：所有图表无立体效果、无渐变背景、无多余边框；网格线一律虚线深灰；轴线/刻度文字一律 muted 灰；数据本身只用一个高亮色（黄）承担。

## 7. 交互状态

截图为静态图，可直接观察到的交互暗示：

- **状态徽章系统**：In progress（黄底黑字）/ Review（橙底白字）/ Complete（绿底白字）/ Blocked（红底白字）——颜色即状态语义，表格与环形图图例共用同一套色。
- **趋势方向**：绿 = 向好、红 = 恶化，KPI 卡趋势文字与图表徽章一致。
- **可点击元素的可视区分**：黄色实心按钮（New job / YB）= 主行动；深灰框（⌘K Command / 搜索框 / All statuses）= 次级控件；纯文字（Export）= 三级操作。
- **悬浮控件**：页脚中央 `Application ⌄` 深色胶囊、右下角圆形 `i` 帮助按钮 —— 全局工具入口悬浮于内容之上。
- 页面右侧可见原生滚动条，说明为长页滚动浏览场景。

## 8. 按钮样式

| 类型 | 实例 | 样式 |
|---|---|---|
| 主按钮（Primary） | `New job` | 亮黄实心、黑色文字、小圆角矩形 —— 全页最高视觉权重 |
| 头像/用户按钮 | `YB` | 同样黄色实心方块，黑字缩写，与主按钮同族 |
| 次按钮（Secondary） | `⌘K Command` | 深灰底 + 灰白文字，带快捷键提示 |
| 输入框 | `Search jobs, crews.` | 深灰底、灰色占位文字、无外框感 |
| 筛选框 | `All statuses` | 透明/深底 + 细边框，弱于搜索框 |
| 文字按钮 | `Export` | 无边框灰白文字 |
| 悬浮胶囊 | `Application ⌄` | 深色胶囊 + 白字 |
| 图标按钮 | `≡` 菜单、圆形 `i` | 深色方块/圆形底 + 灰白图标 |

规律：**黄色专属于"发起行动"**，其余按钮全部退为深灰系，层级完全靠底色明度区分。

## 9. 组件复用方式

- **面板组件**：KPI 卡 / 整宽图表 / 1/3 面板 / 表格容器是同一组件的四种宽度变体（同底色、同圆角、同内边距逻辑）。
- **"大数字 + 副标题 + 右上徽章"头版**：主图表、次图表、Material spend 面板三处完全同构，是纯复用的区块头。
- **趋势表达**：KPI 卡底部趋势文字 ≈ 图表右上徽章，同一语义的两种形态（文字链 vs 徽章）。
- **状态色板**：环形图图例四色 = 表格状态徽章四色（黄/橙/绿/红），全页一致。
- **进度条行**：Crew utilization 四行是同一行组件的重复。
- **图例行**：色块 + 标签 + 数值，环形图与柱状图底部图例同构。

结论：整页大约只由 6–7 个基础组件（面板、头版、徽章、状态标签、进度条行、图例行、按钮）拼装而成，复用度极高，非常适合组件化前端实现。

## 10. 页面节奏

- **纵向节奏**：导航（矮）→ KPI（中矮）→ 主图表（高）→ 次图表（中高）→ 三联面板（中）→ 表格（高），呈"矮-高-高-中-高"的呼吸感；两块整宽大图之间没有并排，保证每个重点独占一行。
- **横向节奏**：4 等分 → 1 整行 → 1 整行 → 3 等分 → 1 整行，分栏数 4→1→1→3→1，避免单调。
- **间距**：结构带之间间距统一且慷慨（相对 1750 宽画布约 24–40px），卡片内部留白充足，深色界面靠留白而非描边分区。
- **视觉重量分布**：最重的颜色（亮黄）集中在页面中段主图与顶部按钮，首尾以灰白文字为主，阅读动线自然被引导到核心图表。

## 11. 可迁移到 HTX OTC BD Progress Hub 的设计原则

### 11.1 结构迁移映射

### 11.2 可直接继承的原则

1. **深色指挥台骨架不变**：`--bg-main:#111111` 页面底、`#1f1f1f` 面板、`#252525` 卡片、`#333333` 分隔，白/灰双级文字。
2. **单高亮色纪律**：黄色 `#ffe000` 保留为**唯一高亮行动色**——主按钮、关键图表主数据、当前主攻项高亮，全页不再出现第二个高饱和装饰色。
3. **HTX 蓝作为业务识别色**：在参考图基础上**新增** `--accent-blue:#0066ff`，仅用于 HTX 业务识别与"进行中（Doing）"状态——对应参考图中"近黑次系列"的位置（如柱状图第二系列、状态徽章 Doing、品牌标识位），不抢夺黄色的主焦点地位。
4. **状态语义全局统一**：Done=绿 `#10b981`、Doing=蓝 `#0066ff`、Next=灰、Blocked=红 `#ff4d6d`——徽章、图例、KPI 卡、看板列四处必须同源同色（参考图的"状态色板复用"手法）。
5. **"大数字结论 + 右上趋势徽章 + 虚线目标线"三件套**：每个核心图表沿用此头版结构；虚线目标线用于表达 PIP 目标，是上级一眼判断"进度是否达标"的关键。
6. **组件化实现**：面板、头版、徽章、状态标签、进度条行、图例行 6 个基础组件先行，全页拼装；宽度变体仅 1/4、1/3、整宽三种。
7. **信息降噪**：说明文字一律灰色小字；表格内用"灰大写（编号/模块）vs 白（关键值）"做行内分层；无渐变背景、无蓝紫配色、无立体阴影。
8. **页面节奏**：先 KPI 总览 → 本周进展主图 → 趋势 vs 目标 → Pipeline 明细 → CRM 漏斗 → 阻塞与下周计划；重点区块独占一行，间距统一慷慨。
