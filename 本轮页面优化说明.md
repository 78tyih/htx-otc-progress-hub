## 2. 收敛逻辑（为什么删）

- **删设计交付清单独立区块**：该区块与任务树 T001「设计交付包」8 项子任务、KPI「设计交付包」进度环（8/8）三处信息完全同源。同一事实在三处重复展示，既增加每周维护成本（要改三个地方），也稀释扫读注意力。保留「任务树（过程明细）+ KPI 环（结果完备度）」两个互补视角，删除纯列表式重复。
- **删本周柱状图**：柱状图表达「本周每天做了多少」，与甘特图（任务何时做、跨哪几周）和执行摘要「本周完成 x/y」卡（结果计数）功能重叠；且「按天统计工作量」对上级汇报价值低——上级关心的是节点与风险，不是每日产出柱高。
- **下周计划并入里程碑**：「下一节点」与「下周要做的事」本质是同一条时间线的两种粒度，拆成两个区块会被误读为两套计划。合并为「关键里程碑（时间线）+ 下周计划（本周行动）」左右双栏后，时间与行动一屏对应。
- **KPI 8 卡收敛为 6 卡**：Partner / KOL / 销售对接本是同一条「渠道拓展」主线的三个子指标，平铺三卡是罗列、合成多轨条才是结构；收敛后一屏看全 PIP 六大目标，且每卡组件形态与指标性质一一对应（见第 4 节）。

## 3. 布局优化逻辑

- **12 栅格**：KPI / Workstream / 执行摘要卡区均采用响应式栅格（宽屏 3 列 → ≤1200px 2 列 → ≤560px 1 列），卡片宽度由栅格决定而非写死。
- **clamp 字号**：主标题等关键字号用 `clamp()` 流式缩放（如 h1 `clamp(17px, 2vw, 21px)`），介于断点之间的宽度不再有字号跳变。
- **两线标题**：区块标题统一「中文主标题 + 一行副标题（说明口径 / 数据源）」两线结构，主标题回答「这是什么」，副标题回答「数据怎么算 / 怎么交互」。
- **卡片底部对齐**：KPI / 摘要 / Workstream 卡均用 flex 列布局把「下一步」行压到卡片底部，同排卡片高度不一致时行动信息仍然齐平。
- **间距收紧**：顶栏、区块间距、卡片内边距整体收紧一档，甘特行高压缩，一屏信息密度显著提升，适配「30 秒扫读」场景。
- **横向防溢出**：`body { overflow-x: hidden }` + 各卡片 `min-width: 0` / `overflow: hidden` 防线；甘特单独 `overflow-x: auto` 容器内滚动。QA 实测 390px 时发现顶栏标题（nowrap）溢出 220px，已最小修复（`.title-block` 加 `min-width: 0`，h1 加省略号截断），修复后 390 / 560 / 768 / 1280px 四档均无页面级横向滚动。

## 4. 动态组件设计逻辑（component × 指标性质）

| KPI | component | 指标性质 | 选型理由 |
| --- | --- | --- | --- |
| 机构注册 | segments 分段格 | 枚举型（6 家逐个落定） | 一家一格、落定点亮，「还差几家」可数 |
| 个人注册 | water 水位 | 累积型（0→19 人逐渐涨） | 水箱波纹水位最直觉表达「蓄了多少」，空箱紧迫感最强 |
| 交易收入 | money 金额条 | 金额型（USDT 增长） | 发光液态条 + 千分位数字滚动，突出「钱在累积」 |
| 设计交付包 | ring 进度环 | 完备型（8 项齐不齐） | 环形表达「圆是否闭合」，8/8 收环即完工 |
| 客户Pipeline | funnel 迷你漏斗 | 转化型（逐级流转） | 复用 CRM 漏斗 7 阶段数据，一眼看客群分布形态 |
| 渠道拓展 | multi 多轨条 | 多线并计型（3 条线合计） | Partner / KOL / 销售三轨并列 + 合计数，结构与总数兼得 |

统一约束：所有组件先落终值再播动画；数值一律以文本为准（如水箱保留 5% 底水只为呈现波纹，真实百分比看标签）。

## 5. 命名规则说明（Simon 抽象化）

- 页面、文档、数据、变量、文案中，汇报对象一律记为 **`Simon`**，禁止真实姓名、亲昵称呼与拼音变体。

- 占位联系方式统一 `simon@example.com` / 用户名 `simon`（仅出现在 `.env.example`、文档示例中），真实邮箱只存在于 Cloudflare Access 后台与本地 `.env`，永不入库（见 `SECURITY.md`）。
